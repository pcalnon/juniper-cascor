## Root Cause Analysis

### 1. Lockfile Freshness Failure (Two Issues)

#### 1a. Spurious comment in `requirements.lock`

The committed `requirements.lock` contains a manually-added line:

```
# lets make this file different
```

This line appears at the end of the file (before `# torch`) and is not produced by `uv pip compile`. The CI lockfile freshness check regenerates the lockfile to a temp file and diffs it against the committed version.

**Evidence** (from CI diff output):
```diff
@@ -176,5 +176,4 @@
 # The following packages were excluded from the output:
-# lets make this file different
 # torch
```

#### 1b. Header comparison bug in CI check (structural — affects main too)

The lockfile freshness check (`.github/workflows/ci.yml`, lines 526-541) generates a temp lockfile with `-o /tmp/requirements.lock.check` and compares it against the committed `requirements.lock` using `diff -u`. However, `uv pip compile` embeds the output path in the file's header comment:

- Committed file header: `#    uv pip compile ... -o requirements.lock`
- CI-generated file header: `#    uv pip compile ... -o /tmp/requirements.lock.check`

This means the diff will **always** show a difference on line 2, causing the check to fail even with a perfectly valid lockfile. This bug also causes lockfile freshness checks to fail on `main`.

**Evidence** (from CI diff output):
```diff
@@ -1,5 +1,5 @@
 # This file was autogenerated by uv via the following command:
-#    uv pip compile ... -o requirements.lock
+#    uv pip compile ... -o /tmp/requirements.lock.check
```

### 2. Unit Tests + Coverage (Python 3.11) Failure

The CI workflow (`.github/workflows/ci.yml`, line 142) defines a test matrix that includes Python 3.11:

```yaml
matrix:
  python-version: ["3.11", "3.12", "3.13"]
```

However, `pyproject.toml` declares:

```toml
requires-python = ">=3.12"
```

When the CI job attempts `pip install` on Python 3.11.14, installation fails immediately:

```
ERROR: Package 'juniper-cascor' requires a different Python: 3.11.14 not in '>=3.12'
```

No tests run; coverage artifacts are not generated.

### 3. Quality Gate Failure (Cascading)

The Quality Gate job aggregates results from upstream jobs. It fails because both the Unit Tests and Lockfile Freshness checks failed. No independent fix is needed — resolving the above will resolve this.

## Fix Plan

### Fix 1: Regenerate `requirements.lock`

**File**: `requirements.lock`
**Action**: Regenerate the lockfile using the canonical `uv pip compile` command to eliminate the spurious comment line.

```bash
cd /home/pcalnon/Development/python/Juniper/juniper-cascor
uv pip compile pyproject.toml \
  --extra ml \
  --extra api \
  --extra observability \
  --extra juniper-data \
  --index-strategy unsafe-best-match \
  --no-emit-package torch \
  -o requirements.lock
```

### Fix 2: Fix lockfile freshness check header comparison bug

**File**: `.github/workflows/ci.yml` (lockfile-check job, lines 535-538)
**Action**: Skip the first 2 lines (auto-generated header comment) when comparing lockfiles, since the output path embedded in the header will always differ between the committed file and the CI-generated temp file.

```yaml
# Before
if ! diff -u requirements.lock /tmp/requirements.lock.check; then

# After
if ! diff -u <(tail -n +3 requirements.lock) <(tail -n +3 /tmp/requirements.lock.check); then
```

This uses process substitution to strip the 2-line header (`# This file was autogenerated...` and `#    uv pip compile...`) from both files before comparing.

### Fix 3: Remove Python 3.11 from CI test matrix

**File**: `.github/workflows/ci.yml`
**Action**: Remove `"3.11"` from the `python-version` matrix in both locations:

1. **Pre-commit job** (line 70):
   ```yaml
   # Before
   python-version: ["3.11","3.12","3.13"]
   # After
   python-version: ["3.12","3.13"]
   ```

2. **Unit Tests + Coverage job** (line 142):
   ```yaml
   # Before
   python-version: ["3.11", "3.12", "3.13"]
   # After
   python-version: ["3.12", "3.13"]
   ```

**Rationale**: `pyproject.toml` declares `requires-python = ">=3.12"` and `tool.black.target-version = ["py312", "py313"]`. No other workflow references Python 3.11. No downstream jobs depend on 3.11-specific matrix outputs.

### Fix 4: Quality Gate (no action needed)

Resolving Fixes 1-3 will cause the Quality Gate to pass. No direct changes required.
