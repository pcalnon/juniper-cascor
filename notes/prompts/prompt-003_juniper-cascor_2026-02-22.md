# PyPi Project Recommendations

## Description

- research, analyze, and recommend how best to organize juniper related project, repos, and packages on pypi.
- Current organizational structure:
  - there is a single Juniper Project
  - there are multiple repos within the project
  - some repos contain packages
  - other repos contain services
- new components that will be added to the juniper project, should fall into these categories:
  - juniper application services
  - juniper packages

## Directives

- this is a research, design, and planning task.
- there should NOT be ANY changes to any of the Juniper project related applications' code bases.

## Task List

- consider pypi documentation, online sources, tutorials, and best practices to recommend an organizational approach.
- if applicable, provide alternate suggested approaches with their relative strengths and weaknesses explained.
- include the research and analysis that went into this (or these) recommendation(s).

## Deliverables

- write up all research, resources, analysis, and recommendations into a JuniperCascor/juniper_cascor/notes/PYPI_PROJECT_STRUCTURE.md document.
